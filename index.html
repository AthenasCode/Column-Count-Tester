About

On older versions of Chrome, CSS's column-count property behaves differently. If you are on a newer version of Chrome, you may not see a difference between the below columns. However, if you open this website in an older version of Chrome, you will see the first two sections have misaligned columns. This is because in older versions of Chrome, the margin of the bottom item of the first column will separate from it's block and shift to the top of the second column.

Universal styles for all blocks:

.block { border: 1px solid black; width: 100%; box-sizing: border-box; margin-bottom: 10px; break-inside: avoid-column; }

Block vs flex vs inline-block

Blockstarts on a new line and extends to the full width of its container, pushing subsequent elements to a new line. Inline-blockallows elements to sit next to each other horizontally, like inline elements, but you can still set width, height, and vertical margins like block elements. Flexitem margins behave similarly to block, depending on the container's axis.
